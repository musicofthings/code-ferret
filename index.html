CodeFerret

An autonomous code review and bug-hunting plugin for Claude Code. It reviews your diff — not your whole repo — and only reports bugs it can prove.

Five detection vectors, zero style nitpicks

CodeFerret targets deep semantic flaws that compilers and linters miss. Anything a linter already enforces is deduplicated away, and every finding carries a confidence rating backed by a concrete failure scenario.

LOGIC

Run-time integrity

Off-by-one errors, boundary conditions, null/undefined flow, unhandled promises, resource leaks, infinite loops.

SECURITY

Vulnerability trapping

Hardcoded secrets, SQL injection, XSS, unsafe deserialization, OWASP Top 10 patterns.

CONCURRENCY

Asynchronous safety

Race conditions, deadlocks, non-atomic read-modify-write, unsynchronized shared state.

PERFORMANCE

Resource conservation

O(N²) loops over unbounded data, N+1 query patterns, redundant allocations.

API

Contract enforcement

Breaking public API changes, type-safety violations, third-party SDK misuse.

NOISE CONTROL

False-positive cache

Dismiss a finding once and its structural pattern is suppressed in every future review. Commit the cache to share with your team.

Install

Requires Claude Code. Installation takes under a minute — run these inside any Claude Code session.

  1. Add the marketplace

    Registers this GitHub repository as a plugin source.

    /plugin marketplace add musicofthings/code-ferret

  2. Install the plugin

    /plugin install code-ferret@code-ferret-marketplace

  3. Review your changes

    Open a new session in any git repository and run:

    /code-ferret:review          # uncommitted changes vs HEAD
/code-ferret:review staged   # staged changes only
/code-ferret:review main     # everything on your branch vs main

Commands

CommandWhat it does
/code-ferret:reviewFull semantic review of the diff. Prints a report with clickable file:line:col locations, severity and confidence per finding, and ready-to-apply patches. Findings are saved to .ferret/last-review.json.
/code-ferret:triageSteps through findings one by one: Accept & apply patch, Ignore pattern (suppresses it forever), Discuss, or Skip.
/code-ferret:precommitFast staged-only check for commit blockers — secrets and high-confidence critical bugs. Verdict is a single FERRET: PASS or FERRET: BLOCK line.
Built-in commit guard: once installed, a hook scans staged changes for credentials (GitHub, AWS, Anthropic, Slack tokens, private keys) whenever Claude Code runs git commit — and blocks the commit if any are found. No LLM call, no latency. Add a .ferretignore file (gitignore syntax) to exclude generated files from review.

Verified test results

Before release, the installed plugin was tested end-to-end against a repository seeded with three real bugs — an SQL injection, a balance-debit race condition, and an off-by-one crash. It found all three:

[CRITICAL · SECURITY · HIGH] src/app.py:2:23
  SQL injection via f-string query construction
  Input like x' OR '1'='1 returns arbitrary rows. Previous version was parameterized.

[CRITICAL · CONCURRENCY · HIGH] src/app.py:6:15
  Race condition in balance debit read-modify-write
  Two concurrent debits both read the same balance; one deduction is silently lost.

[CRITICAL · LOGIC · HIGH] src/app.py:11:9
  Off-by-one causes IndexError in last_items
  range(…, len(items)+1) indexes items[len(items)] — crashes on every call.

Beyond your terminal

The same review engine runs in CI and at the edge — both optional.

GitHub Action

Copy examples/github-workflow.yml into .github/workflows/, set the ANTHROPIC_API_KEY secret, and every pull request gets inline review comments automatically.

Cloudflare Worker proxy

A zero-retention webhook middleware: verifies GitHub signatures, scrubs secrets to [REDACTED_SECRET] before anything reaches the LLM, posts inline PR comments, and reports Check Runs. Deploy from worker/ with wrangler deploy.

Native git hook

Commit outside Claude Code? Install examples/git-pre-commit-hook as .git/hooks/pre-commit for the same secret guard in plain git.
