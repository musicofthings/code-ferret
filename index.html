CodeFerret is an autonomous code-review and bug-hunting plugin for Claude Code. It reviews the changes you actually made — not your whole repo — and only reports issues it can back with a concrete failure scenario.

/code-ferret:review

Deep, diff-scoped semantic review

Point CodeFerret at your changes and it reads them like a detective — tracing suspicious lines across five detection vectors and reporting only what it can prove.

  • Clickable file:line:col locations, with severity and confidence per finding
  • Ready-to-apply patches, each checked with git apply --check
  • Findings saved to .ferret/last-review.json
# uncommitted changes vs HEAD
/code-ferret:review
/code-ferret:review staged   # staged only
/code-ferret:review main     # branch vs main

Five detection vectors, zero style nitpicks

CodeFerret targets deep semantic flaws that compilers and linters miss. Anything a linter already enforces is deduplicated away.

LOGIC

Run-time integrity

Off-by-one errors, boundary conditions, null/undefined flow, unhandled promises, resource leaks, infinite loops.

SECURITY

Vulnerability trapping

Hardcoded secrets, SQL injection, XSS, unsafe deserialization, OWASP Top 10 patterns.

CONCURRENCY

Asynchronous safety

Race conditions, deadlocks, non-atomic read-modify-write, unsynchronized shared state.

PERFORMANCE

Resource conservation

O(N²) loops over unbounded data, N+1 query patterns, redundant allocations.

API

Contract enforcement

Breaking public API changes, type-safety violations, third-party SDK misuse.

NOISE CONTROL

False-positive cache

Dismiss a finding once and its structural pattern is suppressed in every future review. Commit the cache to share it with your team.

/code-ferret:triage

Walk your findings, one at a time

Triage steps through each finding and hands you the decision — nothing gets applied behind your back.

  • Accept & apply patch — applies the fix and confirms it compiles
  • Ignore pattern — suppresses that structural pattern forever
  • Discuss or Skip — keep control of every call
/code-ferret:precommit

A fast gate before every commit

A staged-only check for commit blockers — secrets and high-confidence critical bugs — with a single-line verdict.

  • Returns one clear FERRET: PASS or FERRET: BLOCK
  • Built-in guard scans staged changes for GitHub, AWS, Anthropic & Slack tokens and private keys
  • Pure pattern match on git commit — no LLM call, no latency

agents/ferret-reviewer

A dedicated reviewer sub-agent

Reviews run through a focused ferret-reviewer agent that keeps analysis scoped and consistent across every detection vector — even on large, sprawling diffs.

  • Security, correctness, concurrency and performance evaluated together
  • Findings consolidated into one structured report
  • Deterministic output schema for downstream tooling
Commit guard

Secrets never reach the remote

Once installed, a hook scans staged changes for credentials whenever Claude Code runs git commit — and blocks the commit if any turn up.

$ git commit -m "add config"
CodeFerret blocked this commit: credentials in staged changes.
SECRET src/config.py:1
  • Catches hard-coded keys before they're pushed — no LLM call
  • Add a .ferretignore (gitignore syntax) to exclude generated files
Beyond your terminal

The same engine runs in CI and at the edge

Both optional — take review from your laptop to every pull request.

  • GitHub Action — drop examples/github-workflow.yml in, set ANTHROPIC_API_KEY, get inline PR comments
  • Cloudflare Worker proxy — verifies signatures and scrubs secrets to [REDACTED_SECRET] before anything reaches the LLM
  • Native git hook — install examples/git-pre-commit-hook for the same guard in plain git
Install

Up and running in under a minute

Requires Claude Code. Run these inside any Claude Code session.

  1. Add the marketplace

    /plugin marketplace add musicofthings/code-ferret

  2. Install the plugin

    /plugin install code-ferret@code-ferret-marketplace

  3. Review your changes

    /code-ferret:review
